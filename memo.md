# 設計

## どんなコンテナが必要か

- ブラウザからのリクエストを受け取るのは誰か？
- PHP（Laravel）のコードを実行するのは誰か？
- データを永続化するのは誰か？

# コンテナ構成の整理

```
コンテナ名	      役割
nginx	        ブラウザからの HTTP リクエスト受け口
app（PHP-FPM）	 Laravel のコード実行環境
mysql	        データの永続化
```

# 設計条件

① コード変更を即反映させるにはどうすればよいか？

ローカルで編集したファイルがコンテナ内にも反映される仕組みを Docker のどの機能で実現できるか考えてみましょう。

② コンテナを再起動してもDBのデータを失わないにはどうするか？

コンテナが削除・再作成されてもデータが消えない仕組みを考えてみましょう。

③ コンテナ同士はどうやって通信するのか？

Nginx が PHP-FPM へリクエストを転送するとき、「どこに転送するか」をどう指定するか考えてみましょう。
